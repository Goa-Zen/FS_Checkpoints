3. 1. ¿Qué diferencia a Javascript de cualquier otro lenguaje de programación?

JavaScript es el único lenguaje de programación que realmente puede ser entendido por un navegador web.

Fue creado para ser el único lenguaje que un navegador realmente podría entender, lo que significa es que puede escribir código JavaScript, simplemente abrirlo en el navegador, y ese navegador puede analizar el código, interpretarlo y luego ejecutar el programa.

Además JavaScript es una de las herramientas más poderosas que puedes tener para automatizar tu flujo de trabajo diario.

Finalmente, actualmente si deseamos crear aplicaciones móviles, Javascript permite crear una aplicación utilizando nada más que JavaScript.

Eso significa que se puede crear una aplicación que pueda usar todas las cosas para las que antes necesitabas aprender un idioma completamente diferente con Javascript.

Hoy en día es uno de los lenguajes más utilizados por las razones expuestas.

3. 2. ¿Cuáles son algunos tipos de datos JS?

En JavaScript, tenemos valores primitivos y valores de referencia. Number, Boolean, Null, Undefined, String y Symbol son valores primitivos, mientras que los objetos como nodos DOM, arrays, etc., son valores de referencia.

Valores Primitivos

  • Boolean: Puede adoptar dos valores, "true" or "false".
  • Null: No es lo mismo que un vacio ('') o un espacio en blanco (' '). Es que es nulo.
  • Undefined: Es una variable con un valor no definido.
  • Number: Acepta valores numéricos de todo tipo, enteros, decimales, etc,..
  • String: Acepta letras y números pero los número los trata como textos.
  • Symbol: Es un tipo especial de datos que representaría un símbolo.

Veamos ejemplos de cada uno de ellos: 


                // Boolean
                var verdad = true;
                var no_verdad = false;
            
                // Null
                var nulo = null;
            
                // Undefined
                var notDefined;
            
                // Number
                var kms = 12;
            
                // String
                var nombre = "Alba";
                var apellido = 'Salaberria';
            
                // Symbol
                var simbolo = Symbol('foo');

Valores de referencia

Y en Javascript tenemos los denominados "Objects" también:

Por ejemplo, creamos un objeto que represente a un libro, este libro tendrá unas características, como un author, el nombre del libro, un número de páginas, el nombre de la editorial que la publicó,.. etc

Este libro además tendrá metodos, como abrir el libro, cerrarlo, pasar página.

Por lo tanto, los objetos en JavaScript son estructuras de datos fundamentales que comprenden propiedades y métodos. Mientras que los métodos son funciones/acciones que un objeto puede realizar, las propiedades son características de un objeto.

Los objetos te permiten agrupar datos relacionados y dividir el código en piezas lógicas. 


            var book = {
                name: 'Fiesta',
                pages: 152,
                author: 'Ernest Hemingway',
                publishers: {
                    alianza: 90,
                    Bruguera: 80,
                    Alfaguara: 100
                }
            }

El libro en este caso, además de tener las propiedades "name","àges" y "author" tiene una propiedad anidada que recoge los diferentes ejemplares disponibles en cada Editorial.

3. 3. ¿Cuáles son las tres funciones de String en JS?

Para empezar con este tema, vamos a definir una variable string sobre la que aplicaremos todos los ejemplos citados: 


                var str = 'El veloz murciélago hindú comía feliz cardillo y kiwi';

Y antes de proseguir vamos a advertir sobre dos cosas: 1- La longitud de una variable, "length" no es una función, es una propiedad. 2- Cuando hablamos de "posición", hay que considerar que para JS todo empieza a contar de cero.

Si hemos de dividir los tipos de funciones en JavaScript en tres, serían las siguientes:

  1. Funciones de cadenas para realizar tareas como buscar valores, encontrar el índice de un carácter y otros.
    • str.charAt(number) -- > Output: caracter en la posición indicada en "number".
    • str.concat(cadena) -- > Output: Devuelve una cadena con los dos valores concatenados.
    • str.includes(cadena) -- > Output: Devuelve true/fase.
    • str.endsWith(cadena) -- > Output: Devuelve true/fase.
    • str.startsWith(cadena) -- > Output: Devuelve true/fase.
  2. Funciones que usan tanto cadenas como expresiones regulares para encontrar patrones de datos en cadenas.
    • str.match(cadena/RegExp) -- > Output: Si lo encuentra nos devuelve un array con los valores, si no encuentra nada devuelve null.
    • str.replace(cadena_search/RegExp_search, cadena_replace/regular_expr_replace) -- > Output: Devuelve la cadena inicial con las cadenas solicitadas reemplazadas.
    • str.search(cadena/RegExp) -- > Output: Devuelve la primera posición donde se encuentra esa regular expresion.
    • str.indexOf(cadena) -- > Output: Devuelve la primera posición donde se encuentra esa cadena.
    • str.lastIndexOf(cadena) -- > Output: Devuelve la última posición donde se encuentra esa cadena.
  3. Funciones "slice", "case" y "trim" y ver como podemos encadenar cadenas.
    • str.slice(start_position, end_position -optional) -- > Output: Devuelve los caracteres que se encuentran entre la start_position, hasta la end_position. Si esta end_position no se determina, por defecto iría hasta el final. ".
    • str.toUpperCase() -- > Output: Devuelve el valor de la variable str convertido a mayúsculas.
    • str.toLowerCase() -- > Output: Devuelve el valor de la variable str convertido a minúsculas.
    • str.trim() -- > Output: Devuelve el valor de la variable str sin espacios delante ni detrás.

3. 4. ¿Qué es un condicional?

Un condicional permite introducir todo tipo de condiciones de forma que la ejecución sea dinámica ya que así permitimos al sistema tomar decisiones.

Es decir, sin los condicionales lo que programaramos sería lineal y no habría opciones de tomar caminos diferentes en función de un suceso o evento creado por el usuario.

Nosotos cuando hemos desarrollado el código para el ejercicio hemos hecho uso de dos tipos de condicionales:

  • "if ,.. else" - "if ,.. else if ,.. else ,.."
  • switch

"IF ,.. ELSE" - "IF ,.. ELSE IF ,.. ELSE ,.." 


                var name = 'Alba';
                var response = null;                        
                if (name === 'Alba'){
                    response = 'You are who I spect.';
                } else {
                    response = 'You are not who I spect.';
                }

El condicional anterior lo que hace es ver si el nombre es "Alba" y entonces si es así, devuelve en la variable response el texto . Y si no es así, entonces, devolvería 'You are not who I spect.'

Y esto podemos hacerlo tan complejo como queramos:


                var name = 'Alba';
                var response = null;
                if (name === 'Alba' || name === 'Rocio'){
                    response = 'You are who I spect.';
                } else if(name === 'Asier' ){
                    response = 'You are not allowed to be here!!';            
                } else {
                    response = 'You are not who I spect.';
                }
Operadores compuestos

Acabamos de hacer uso de "||" que significa "or".

Y podríamos también de hacer uso de "&&" que significa "and".

Operadores de los condicionales

Cuando escribimos una condición, los operadores de condición no se corresponden exactamente a los operadores de asignación en lo que se refiere al operador "=".

En el caso de ser un operador de una condición, tal y como hemos visto, utilizaríamos "===".

En este caso, como se puede observar hemos asignado el mensaje a una variable mediante el operador de asignación "=" y sin embargo para comprobar la condición hemos utilizado el operador condicional "==".

Los operadores Condicionales son los siguientes:

  • === Equality
  • !== Inequality
  • > Greater than
  • >= Greater than or equal to
  • < Less than
  • <= Less than or equal to

3.5. ¿Qué es un operador ternario?

Es posible simplificar una condición implementando lo que se denomina ternary operator.

Estas dos condiciones serían la misma siendo la segunda más breve como se puede observar.


    
                var name = 'Alba';
                var response = null;

                // Ordinary conditional
                if (name === 'Alba'){
                    response = 'You are who I spect.';
                } else {
                    response  = 'You are not who I spect.';
                }
                    
            
                // Conditional using Ternary operator
                response =  if name === 'Alba' ? 'You are who I spect.' : 'You are not who I spect.';

Podemos ver que la condición que poníamos en el if, es la condición que anteponemos a la "?" en el Ternary operator: El valor tras la "?" es el valor en caso de que sea "true" la condición, y en segundo lugar, el valor en caso de que no sea true la condición.

Podemos utilizar operadores ternarios para dos condiciones también, pero esto resulta más incómodo de leer. : 


                // Ordinary conditional
                function foodControl(article) {
                    if (article) {
                        if (article.veg) {
                            return 'it is a veg.';
                        } else {
                            return 'it is not a veg.';
                        }
                    } else {
                        return 'you need to be some kind of foood.';
                    }
                }
            
            
                let food = {
                    name: 'Broccoli',
                    veg: true
                }
                let output= foodControls(food);
            
            // Conditional using Ternary operator
            let output = food ? food.veg ? 'it is a veg.' :'it is not a veg.': 'you need to be some kind of foood.';
            console.log(output);

Con "{" sería lo mismo pero más claro:

let output = food ? {food.veg ? 'it is a veg.' :'it is not a veg.':} 'you need to be some kind of foood.'; console.log(output);
