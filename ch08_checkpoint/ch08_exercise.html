3. 1. ¿Qué tipo de bucles hay en JS?

Los bucles son instrucciones que nos permiten recorrer todos los elementos de un array, uno por uno, o mientras que se de una condición.

Los bucles son absolutamente necesarios. Aunque existen muchas funciones que operan sobre todo tipo de objetos, a veces necesitamos ser capaces de recorrer los elementos para operar sobre ellos. Sin los bucles, esto sería imposible y por tanto sería imposible implementar algoritmos complejos o dinamismo a una aplicación.

Existen fundamentalmante dos tipos:

  1. for
  2. while

for recorre los elementos, desde un elemento inicial hasta otro final que determinamos o viene determinado por la propia estructura de la colección que recorremos.

while recorre un elemento hasta que se produce una situación donde le decimos que deje de recorrerlo. OJO! Es importante al definir un bucle "while" determinar una instrucción que nos impida entrar en un bucle "infinito".

2. while

Existen 2 tipos:

  • while()
  • do {} while()

while()

Empezamos por la instrucción donde le decimos "Mientras que se de esta situación", en este caso sería que la suma de los valores sea menor que el valor máximo ("max_value_sum") y que el número de elementos en la lista de salida "list_of_values", sea menor a la longitud determinada "len", que en este caso es 10. 

    
function goAcrossWhile () {
    try {
        let len = 10;
        let max_value_sum = 100;
        let ratio = Number(max_value_sum/len);
        let sum_of_values = 0;
        let list_of_values = [];
        
        while (sum_of_values < max_value_sum && list_of_values.length < Number(len)) {
            let value = Math.round(Math.round(Math.random()* ratio));
            sum_of_values= Number(sum_of_values) + Number(value);
            list_of_values.push(value);
        }

        return list_of_values;  
    }catch (err) {
        document.getElementById('code-error').innerHTML = err.message;
    }
};

do {} while()

Esta instrucción podría parecer idéntica a la anterior, pero si nos fijamos, en esta, suceda la condición que suceda en primer lugar se ejecuta al menos una vez la instrucción contenida en el "do", mientras que en la anterior, al evaluar la condición al principio, si nada más iniciar el bucle no se cumple, no se ejecuta nunca ninguna instrucción. 

    
function goAcrossWhile () {
    try {
        let len = 10;
        let max_value_sum = 100;
        let ratio = Number(max_value_sum/len);
        let sum_of_values = 0;
        let list_of_values = [];

        do {
            value = Math.round(Math.ceil(Math.random()* ratio));
            sum_of_values= Number(sum_of_values) + Number(value);
            list_of_values.push(value);
        } while (sum_of_values < max_value_sum && list_of_values.length < len);

        return list_of_values;  
    }catch (err) {
        document.getElementById('code-error').innerHTML = err.message;
    }
};

3. 2. ¿Cuáles son las diferencias entre const, let y var?

característica
var
let
const
Alcance
Su ámbito es de función local o global. Si es declarada dentro de una función, entonces solo es accedible en esa función.
Si es global, está disponible su uso en general.
Su ámbito es de bloque ({}) y la variable "let" solo está disponible en ese bloque.
Al igual que let solo está accesible dentro del bloque en la que han sido declaradas.
Uso
Las variables con var se pueden volver a declarar y modificar,
Esto significa que podemos hacer esto dentro del mismo ámbito y no obtendremos un error.
Al igual que var, una variable declarada con let puede ser actualizada dentro de su ámbito.
A diferencia de var, una variable let no puede ser re-declarada dentro de su ámbito.
"const" no puede modificarse ni volver a declararse.
Esto significa que el valor de una variable declarada con const es el mismo dentro de su ámbito.
Hoisting
Si no declaramos la variable "var", el sistema interpreta que lo ha declarado como "undefined" , y la inicializa como tal.
A diferencia de var que se inicializa como undefined, la palabra clave let no se inicializa.
Si intentas usar una variable let antes de declararla, obtendrás error.
Al igual que let, const las declaraciones const no se inicializan.

Funciones Flecha

- Arrow Function: una función flecha nos devuelve un valor, que es el que queda almacenado dentro del objeto (como en el caso del ejemplo), o sencillamente devuelve un valor. En principio es como cualquier otra función pero además de estar escrita de forma más "resumida":

  • Las funciones de flecha no tienen this. Si se accede a this, se toma el contexto del exterior.
  • JavaScript ofrece una gran flexibilidad cuando se trata de funciones. Se pueden pasar, usar como objetos, y redirigir las llamadas entre ellas y decorarlas. Como la función flecha no tienen variable "arguments" es ideal para ello.
  • Las variables flecha al no tener this, no pueden ser usadas como constructores, y por tanto no se pueden utilizar con "New".
  • La diferencia entre una función de flecha => y una función regular llamada con .bind(this):
    .bind(this) crea una “versión enlazada” de la función y la función flecha => no crea ningún enlace. La función simplemente no tiene this.
    La búsqueda de ‘this’ se realiza exactamente de la misma manera que una búsqueda de variable regular, en el entorno externo. 

  
function operationsComputer() {
    try {
        this.arguments = [15,30,10,45];

        this.sum = () => {
            let obj = this.arguments;
            let reducer = (accumulator, curr) => accumulator + curr;
            return Number((obj.reduce(reducer))); 
        }
        this.mean = () => {
            let obj = this.arguments;
            let reducer = (accumulator, curr) => accumulator + curr;
            return Number((obj.reduce(reducer)))/Number(obj.length); 
        } 
    }catch (err) {

3. 4. ¿Qué es la deconstrucción de variables?

La sintaxis de deconstrucción es una funcionalidad de JavaScript que permite extraer valores de arreglos o propiedades de objetos en distintas variables. Es decir, podemos extraer datos de arreglos y objectos y asignarlos a variables.

¿Qué es un arreglo en JavaScript?

Todos los elementos en un arreglo están separados por una comma(,). Por ejemlo. Un par de square brackets [] representa un arreglo en JavaScript.

En JavaScript, los arreglos pueden ser una colección de elementos de cualquier tipo, lo que significa que tú puedes crear un arreglo con elementos de tipo Cadena , Boolean, Número, Objetos, e incluso otros Arreglos.

Si declaramos un array, por ejemplo "let taxes = [0.02, 0.20, 0.40, 0.45, 0.50]". Si quisieramos conseguir el primer o segundo elemento, tradicionalmente, hubiéramos llamado a taxes[0] y taxes[1] y hubiéramos conseguido los valores.

Actualmente podemos hacerlo de otra forma, que es mediante la Deconstrucción. 


function desconstruction() {
    try {
        let taxes = [0.02, 0.20, 0.40, 0.45, 0.50];
        let [first,second] = taxes;
        return `La primera tasa es del ${first}% y la segunda del ${second}% `;
    }catch (err) {
        document.getElementById('code-error').innerHTML = err.message;
    }
};

Para hacer una copia

Si hacemos directamente la copia de un objeto, vemos, que si posteriormente añadimos un elemento a ese objeto, se añade, tanto a la copia como al original. Para evitar esto, hacemos la copia usando la función ".slice(posicion_inico, posicion_fin)" y de esa forma quedan como dos objetos independientes. Y en Javascript tenemos los denominados "Objects" también:

Por ejemplo, creamos un objeto que represente a un libro, este libro tendrá unas características, como un author, el nombre del libro, un número de páginas, el nombre de la editorial que la publicó,.. etc

Este libro además tendrá metodos, como abrir el libro, cerrarlo, pasar página.

Por lo tanto, los objetos en JavaScript son estructuras de datos fundamentales que comprenden propiedades y métodos. Mientras que los métodos son funciones/acciones que un objeto puede realizar, las propiedades son características de un objeto.

Los objetos te permiten agrupar datos relacionados y dividir el código en piezas lógicas.

3. 7. ¿Qué es una promesa en JS?

Las promesas en JavaScript sirven para manejar acciones asíncronas de manera estructurada y legible.

Cuando realizas una acción y esta no se completan de inmediato (como llamadas a API, lecturas de archivos o consultas a bases de datos) JavaScript necesita una forma de gestionar estas acciones sin condicionar la ejecución del resto del código.

En el ejemplo que vamos a poner a continuación no es exactamente como se utiliza habitualmente, pero esto nos permitirá hacernos una idea de que cuando usamos una promesa lo que estamos haciendo es manejar las operaciones que han de suceder de forma cronológica. Imaginemos que después de pedir una serie de números aleatorios, estos han de sumar una cifra determinada a priori. Si esto no lo hace, queremos que nos avise. Vamos a usar una promesa. 


function randomValues() {
    try{
        return new Promise((resolve, reject) => {
            const len = 5;
            const max_value_sum = 100;
            var sum_of_values = 0 ;
            var list_of_values = [];

            do {
                let value = Math.round(Math.round(Math.random()* (max_value_sum/len)));
                sum_of_values= Number(sum_of_values) + Number(value);
                list_of_values.push(value);
            } while (sum_of_values < max_value_sum && list_of_values.length < len);   
        
            while (list_of_values.length < Number(len)) {
                list_of_values.push(0);
            }
            let queda = max_value_sum - Number(sum_of_values); 
            if (Math.abs(queda)!==0) {
                reject(`No hemos obtenido la suma máxima deseada, ${max_value_sum}. Hemos obtenido ${sum_of_values} con ${list_of_values}`);
            } else{
                resolve(`Hemos obtenido la suma máxima deseada, ${max_value_sum}. Hemos obtenido ${sum_of_values} con ${list_of_values}`);
            }
        })
    }catch (err) {
        document.getElementById('code-error').innerHTML = err.message;
    }   
};

Podemos ver que el sistema nos indica mediante la llamada a la promesa si se ha cumplido o no la acción que necesitabamos, que en el caso de nuestro código son 5 elementos numéricos aleatorios que sumen 100. Cuando esto se produce o no se produce, nos avisa.

Aunque aqui el método .then() lo utilizamos para obtener el resultado , el resultado de una promesa podría ser pasado como entrada a la siguiente promesa en la cadena ".then(result => siguientePromesa)".

Finalmente implementa en una promesa el método catch para retornar los errores.

Además de hacer el código más legible, las promesas permiten manejar acciones de forma paralela. Para ello existen las instrucciones "Promise.all()", que espera a que todas las promesas se cumplan antes de continuar y "Promise.race()" que resuelve todo en cuanto una de las promesas se cumple.
