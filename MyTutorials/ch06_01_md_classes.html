1. ¿Qué es una clase? ¿Por qué son útiles?

1.1. Introducción

Las clases son mapeadores de objetos, es decir, cuando creamos una clase, creamos un objeto al que asignamos unas propiedades y unos métodos.

Por ejemplo, si lo que queremos crear es un objeto que represente a un libro, este libro tendrá unas características, como un author, el nombre del libro, un número de páginas, el nombre de la editorial que la publicó,.. etc

1.2. ¿Por qué son útiles?

Siguiendo con el ejemplo anterior, diríamos que un libro tiene estas variables o atributos asociados.

    book_name = 'Fiesta'
    book_author = 'Ernest Hemingway'
    book_pages = "150"
    book_publisher = "Pan Books"

Si nosotros quisieramos declarar una clase Book que nos permitiera obtener objetos de esos libros nos ayudaría mucho, ya que todas las clases pueden disponer de data y de un comportamiento. Sigamos y lo veremos claramente.

    class Book:

        def __init__(self, book_name, **kwargs):

Con la delcaración "class Book" hemos declarado una clase y con la siguiente función "def init" es como la iniciamos, y ahí es donde pasamos los valores de los atributos que tendrá ese objeto "Book" que crearemos.

    class Book:

        def __init__(self, book_name, **kwargs):
            self.book_name = book_name
            book_attr = dict(kwargs.items())
            self.book_description = {
                "book_name" : book_name,
                "book_attr" : book_attr
                                }

Los parámetros que pasamos a "init", son los siguientes:

  • Toda función dentro de una clase debera llamar a esa palabra "self" por necesidad, ya que ese "self" representa a cada instancia que llama a esa clase. Ahora veremos que es exactamente una instancia.

  • Como segundo parámetro pasamos "book_name", como un parámetro individual que recoge un solo valor, en este caso el nombre del libro.

  • Finalmente pasamos "**kwarg" que responde a una seríe de argumentos que contienen una clave y un valor, es decir, los argumentos que corresponderían a un diccionario de Python.

En definitiva, init es el que método se ejecuta automáticamente cuando se crea una instancia de una clase.

Con estos parámetros vamos a ver que sucede si declaramos la clase con estos valores:

    first_book = Book(book_name = 'Fiesta',
                    book_author = 'Ernest Hemingway',
                    book_pages = "150",
                    book_publisher = "Pan Books"
                    )

    print(first_book.book_description)

Esto que acabamos de hacer, es instanciar una clase, es decir, hemos introducido unos parámetros y hemos creado un objeto llamado "first_book" con esa clase.

Si declaramos un segundo libro, "second_book" e introducimos otros parámetros, podemos ver como las descripciones que nos devuelve son las propias de cada libro.

    second_book = Book(book_name = 'Romeo and Juliet',
                    book_author = 'Shakespeare',
                    book_pages = "156",
                    book_publisher = "Letra Minúscula"
                    )

    print(second_book.book_description)

El primero nos devolverá: {'book_name': 'Fiesta', 'book_attr': {'book_author': 'Ernest Hemingway', 'book_pages': '150', 'book_publisher': 'Pan Books'}}

Y el segundo nos devolverá: {'book_name': 'Romeo and Juliet', 'book_attr': {'book_author': 'Shakespeare', 'book_pages': '156', 'book_publisher': 'Letra Minúscula'}}

!!! -- En defintiva, Debido a esta capacidad de crear instancias, podemos crear varios objetos simplemente instanciando una clase.

1.3. Palabras reservadas: Doble undersecored Methods o Dunder Methods

"The dobles underscored Methods" o tambien llamados "Dunder methods" , son los siguientes:

  • init, que tal y como hemos dicho es la forma de iniciar una clase.

  • str y repr , suelen ser instrucciones que utilizamos sobre todo para obtener una idea rápida de la clase.

Por ejemplo para la clase Book declarada podríamos poner.

    def __str__(self):
        return f'A nested dictionary with {self.book_name}'
    
    def __repr__(self):
        return f'A nested dictionary with {self.book_name} and the description and {self.book_description}'

Al llamar a str, que lo haríamos de la siguiente manera:

    print(str(first_book))

Nos devolvería: "A nested dictionary with Fiesta".

Al llamar a repr

    print(repr(first_book))

Nos devolvería: "A nested dictionary with Fiesta and the description and {'book_name': 'Fiesta', 'book_attr': {'book_author': 'Ernest Hemingway', 'book_pages': '150', 'book_publisher': 'Pan Books'}}"

  • "iter" : Representa la expresión que da a entender que debemos iterar entre los elementos, lo cual significa que los recorremos.

  • "next": Indica que hay que pasar al siguiente elemento.

   class Iterator:

    def __init__(self, objects_to_iteract):
        self.object_to_iteract = objects_to_iteract
        self.last_object = (len(self.object_to_iteract) - 1)

    def __iter__(self):
        self.n = 0
        return self

    def get_object_num(self, n):
        return self.object_to_iteract[n]
    
    def __next__(self):
        if self.n < self.last_object:
            output = self.get_object_num(self.n)
            self.n += 1
            return output
        elif self.n == self.last_object:
            output = self.get_object_num(self.n)
            self.n = 0
            return output

Para entender este código, vamos a añadir 2 libros más con nuestra clase "Book" .

    it = Iterator(stock_libreria)

1.4. Herencia

Si lo pensamos detenidamente, un libro es un artículo o un producto. Es un artículo o un producto, como unos guantes, un ordenador o un coche, pero sin embargo, tiene unas características diferentes, como las citadas, autor, número de hojas y editorial.

Vamos a crear por tanto una clase "Article" y vamos a modificar nuestra clase "Book" de forma que herede los atributos de esta.

    class Article:

        def __init__(self, item_reference, item_name, item_pvp, item_description):
            self.item_reference = item_reference
            self.item_name = item_name
            self.item_pvp = item_pvp
            self.item_description = item_description

            self.item_main = {
                "item_reference" : item_reference,
                "item_name" : item_name,
                "item_pvp" : item_pvp,
                "item_description" : item_description
            }

        def __str__(self):
            return f"REFERENCIA\t {self.item_reference}\n" \
                f"NOMBRE\t\t {self.item_name}\n" \
                f"PVP\t\t {self.item_pvp}\n" \
                f"DESCRIPCIÓN\t {self.item_description}\n"

Esta vez he hecho un poco diferente esta clase, lo que antes pasábamos como **Kwargs, es decir como una serie de argumentos con clave, ahora solicitamos directamente las 4 variables.

Todo artículo tiene una referencia en un inventario, un nombre, un precio y una descripción.

Y los libros, además de estas características puede tener las que ya hemos citado al inicio, el autor, el número de páginas y la editorial.

Por lo tanto, lo que vamos a hacer es lo siguiente, declarar la clase "Book" pero ahora le pasaremos el parámetro "Article" para que Book adquiera todas las características de Article y además, veremos, le sumaremos alguna más.

    class Book(Article):

        def insert_book_details(self, **kwargs):
            book_attr = dict(kwargs.items())
            self.book_attr = book_attr
            self.item_main['item_atributtes'] = self.book_attr

Y en en el diccionario que ya habíamos creado con "Article" añadimos un nuevo elemento denominado "item_attributtes", que será como cualquier item del diccionario y podremos acceder a sus elementos, que declaramos como un diccionario.

A la hora de ejecutar el código, cambian cosas, ya que primero introducimos los parámetros de la clase padre, "Article" ya que la clase "Book" es heredera y luego, ya que se trata de un libro y tiene otras propiedades además de la de cualquier artículo, esta clase "Book" ejecuta una función propia y actualiza ese primer diccionario que obtenemos en la declaraciónd e clase "self.item_main".

    first_book = Book(item_reference = 'A/B/2024-2000',
                     item_name = 'Fiesta',
                     item_pvp = 17.5,
                     item_description = "Libro de Ernest Hemingway"
                     )
    print(first_book.item_main)
    first_book.insert_book_details(book_name = 'Fiesta',
                                   book_author = 'Ernest Hemingway',
                                   book_pages = "150",
                                   book_publisher = "Pan Books"
                                  )


    second_book = Book(item_reference = 'A/B/2024-2001',
                       item_name = 'Romeo and Juliet',
                       item_pvp = 27.5,
                       item_description = "Libro de W. Shakespeare"
                      )
    print(second_book.item_main)
    second_book.insert_book_details(book_name = 'Romeo and Juliet',
                                    book_author = 'William Shakespeare',
                                    book_pages = "156",
                                    book_publisher = "Letra Minúscula"
                                   )

    print(first_book.item_main)
    print(second_book.item_main)

Cuando imprimimos "first_book.item_main" antes de aplicarle la función "insert_book_details", obtenemos:

{'item_reference': 'A/B/2024-2000', 'item_name': 'Fiesta', 'item_pvp': 17.5, 'item_description': 'Libro de Ernest Hemingway'}

Cuando imprimimos "second_book.item_main" antes de aplicarle la función "insert_book_details", obtenemos:

{'item_reference': 'A/B/2024-2001', 'item_name': 'Romeo and Juliet', 'item_pvp': 27.5, 'item_description': 'Libro de W. Shakespeare'}

Cuando en ambos libros aplicamos la función "insert_book_details", podemos ver que lo que hace es incluir esos argumentos en un nuevo elemento del diccionario inicial, "item_attributes":

{'item_reference': 'A/B/2024-2000', 'item_name': 'Fiesta', 'item_pvp': 17.5, 'item_description': 'Libro de Ernest Hemingway', 'item_atributtes': {'book_name': 'Fiesta', 'book_author': 'Ernest Hemingway', 'book_pages': '150', 'book_publisher': 'Pan Books'}}

{'item_reference': 'A/B/2024-2001', 'item_name': 'Romeo and Juliet', 'item_pvp': 27.5, 'item_description': 'Libro de W. Shakespeare', 'item_atributtes': {'book_name': 'Romeo and Juliet', 'book_author': 'William Shakespeare', 'book_pages': '156', 'book_publisher': 'Letra Minúscula'}}

Cuando declaramos la variable "libreria" e instanciamos la clase Library, entonces, hacemos la misma operación que hacíamos antes, que es ir añadiendo los elementos, en este caso libros a la libreria. La clase "Library" no cambia, pero sien embargo el nombre de la propiedad de las instancias de "Book", ahora se llaman "item_main" (antes era "book_description")

    libreria = Library()
    libreria.add_book(first_book.item_main)
    libreria.add_book(second_book.item_main)

La clase "Iterator" que habíamos generado tampoco cambia en nada.

1.5. Polimorfismo

Si observáis el código del ejercicio anterior resuelto, descubriréis que todos ejecutan la misma función que tienen todas las clases, tanto la padre como las hijas "register", crear un diccionario con la información que tienen los atributos propios de cada usuario -que no son exactamente los mismos, tal y como hemos visto-.

En este caso la única variable que es difernete es la "user_secure_word", pero el resto es exactamente igual (respecto a lo que representan, no a lo que contienen)

Esto es lo que responde a que es el Polimorfismo en las clases de Python, y representa la capacidad de ejecutar las mismas funciones dentro de las clases childs sin que necesariamente tengan que ejecutar la misma funcionalidad que la clase padre o que las clases hermanas.

!!! -- En defintiva, existen muchas formas de implementar las clases, entre otras la ya vista donde además de heredar los atributos de la clase padre, puede ejecutar una función ya definida en la clase padre con diferente funcionalidad en las clases hijas, pese a adoptar en estas el mismo nombre la función

1.6. GET y SET en Python

Hasta ahora hemos estado añadiendo atributos y funciones a las clases, como diríamos, "a la rica ciruela", es decir, alegremente.

¿Por qué digo esto?

En Python, dada la gran flexibilida de programación, sucede algo que no sucede en todos los lenguajes, y es que en principio todos los atributos, objetos, métodos son accesibles, y con eso quiero decir que puede ver el valor - GET- y PUEDES CAMBIARLO -SET-, lo cual podría tener mucha lógica pero no tiene ninguna ya que cualquier usuario podría acceder a una variable que has creado en la clase fruto de un proceso, y ponerle otro valor.

Si recordáis, cuando hemos definido la clase Library:

    class Library:

        def __init__(self):
            self.books_store = {}
            

        def add_book(self, books_store_new):

            self.books_store[books_store_new['book_name']] = books_store_new['book_attr']
            # return self.books_store

        def list_of_books(self):
            lista = []
            print(self.books_store.values())
            for value in self.books_store.values():
                lista.append([value])
            return lista

    stock_libreria = libreria.list_of_books()

Veís que la variable "books_store" es una variable que es de la clase y sin embargo podemos sobre escribirla.

    libreria.books_store ={"nothing": "nothing to show"}

    stock_libreria = libreria.list_of_books()
    print(stock_libreria)

Y entonces lo que obtenemos en "stock_librería" se basa en esa información y para "librería" todo está muy bien pero par nostros, no.

Entonces, ¿Cómo hacemos para "proteger" estas variables?

1.7. Decoradores en Python

Tendríamos dos formas de "proteger" o "restringir" el acceso a esas variables:

  • Podemos poner "_" previo a una variable y significaría que la estamos protegiendo "protected".

  • O bien poner "__" previo a una variable y significaría que la estamos declarando privada "private".

    # Protected
    class Library:

        def __init__(self):
            self._books_store = {}

    # Private
    class Library:

        def __init__(self):
            self.__books_store = {}

Vamos a aprovechar nuestras clases sin herencia ni poliformismo (1.5. Herencia: Ejercicio resuelto) para ver las diferencias entre las acciones que nos permitirían cada una de ellas.
